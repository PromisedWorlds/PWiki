# Contributing

Thank you for your interest in improving the KSA Modding Wiki!

This is a **curated** documentation project. Contributions are welcome via pull requests and will be reviewed before merging.

## How to contribute

1. **Fork** the repository.
2. **Create a branch** for your changes.
3. Run the site locally with:

   ```bash
   pip install mkdocs mkdocs-material
   mkdocs serve
   ```

4. Make your edits in the `docs/` folder.
5. Ensure pages are:
   - Linked in `mkdocs.yml` navigation.
   - Labeled with the KSA version/build they were tested on, where applicable.
6. Commit and push your changes.
7. Open a **pull request** describing:
   - What you changed.
   - Which KSA build/version you tested against.
   - Any potential breaking changes or new behaviors.

## Style guidelines

- Follow the [documentation style guide](docs/meta/style-guide.md).
- Keep examples minimal but correct and copy-pasteable.
- When possible, document both the **"how"** and the **"why"**.

## Reporting issues

If you find a mistake, outdated information, or a missing topic:

- Open an issue with as much detail as possible.
- Include the KSA build/version you are using.
- Provide steps to reproduce or confirm the behavior when relevant.
